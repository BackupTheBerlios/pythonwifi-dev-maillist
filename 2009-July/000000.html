  Now that there is a subscriber to the Python WiFi Dev mailing list, I
guess I should discuss plans in the open. 8-)

  About 2-3 weeks ago, I discovered that Wireless Extensions is deprecated
for Linux WiFi.  The new preferred model is nl80211, using netlink.  I have
been researching nl80211 and development plans for Linux WiFi, as well as
considering how to go forward with Python WiFi.

  What I propose is to release a v0.5 package in the next several weeks
which cleans up any killer bugs, but does not include me adding new features
(or trying to match every WE feature).  I would then begin adding nl80211 to
Python WiFi, so that a v1.0 release would support many (but not all) WE
features and many nl80211 features.  Essentially, without specific feature
or bug requests, I would (probably) not do many more changes to WE, but
would concentrate on nl80211.

  If anyone has a different view on this, I'm happy to listen.
